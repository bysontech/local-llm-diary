# 実装サイクル計画（Cycle 5 / 1〜2日）

## 期間
- （例）2026/02/11 〜 2026/02/12

## 今回のユースケース（縦切り）
1. 暗号化エクスポート
- 設定画面で「暗号化バックアップ」を選ぶ
- パスワード入力 → 「暗号化してエクスポート」
- encrypted-backup-YYYY-MM-DD.json がダウンロードされる
- 中身は平文のentries/bodyを含まない（暗号化されている）

2. 暗号化インポート
- 設定画面で「暗号化バックアップをインポート」
- JSONファイル選択 → パスワード入力 → 「復号して復元」
- entries と settings が復元される
- パスワード間違い時は復元されず、エラー表示される

## ゴール
- 「端末内完結」のまま、バックアップを安全に持ち出せる
- 既存の平文エクスポート/インポートは壊さない（併存）

## やらないこと
- クラウド同期
- ログイン
- 鍵の保存（パスワードを保存しない）
- 高度な鍵管理UI（強度メーター等）
- 暗号化バックアップの自動スケジュール

## 人が判断するポイント
- インポートは上書き（既存データ全削除→復元）を継続
- 暗号方式はWeb標準（WebCrypto）で実装する
- 本文・要約はログに出さない
