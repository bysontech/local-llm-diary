# 実装サイクル計画（Cycle 7 / 1日）

## 期間
- （例）2026/02/13 〜 2026/02/13

## 今回のテーマ
- 「書くだけ」から「振り返れる」へ：週 or 月の振り返り（最小）を追加する  
- 既存の要約（ルールベース）を再利用し、追加の重い分析はしない  
- 外部送信なし・依存追加なし・オフライン前提のまま進める

## ゴール（ユーザー体感）
- 期間を選ぶと、その期間の記録をまとまって見られる
- 「どんな週/月だったか」を思い出せる（要約一覧で十分）
- 既存の作成/編集/削除/検索/PWA/バックアップ機能は壊れない

## やらないこと
- 週/月要約をLLMで生成（MVP外）
- 感情推定・自動タグ付け
- グラフや統計（件数表示程度はOK）
- 期間横断の高度検索
- 依存追加

## 仕様（最小）
### 新規画面：振り返り（Review）
- 一覧/詳細の導線を崩さず、ナビに追加する（例：ヘッダー or メニュー or 設定から遷移）
- 表示内容（最小）
  - 期間の選択（週 or 月）
  - 選択期間のエントリ件数
  - 選択期間のエントリを日付降順で表示
    - 行の表示：日付 + 要約（なければ本文の短縮表示）
    - クリックで既存の詳細画面へ遷移

### 期間選択（どちらかでOK：実装が楽な方）
- 案A：月選択（おすすめ：実装簡単）
  - YYYY-MM を選ぶ → その月の entries を `date` の prefix で絞る
- 案B：週選択
  - その週の開始日〜終了日で絞る（境界計算が必要）

※ PoC優先のため **月選択を第一候補**。週は次サイクルでもよい。

## 受け入れ条件（DoD）
- [ ] Review画面に遷移できる
- [ ] 月（YYYY-MM）を選ぶと該当エントリが表示される
- [ ] 表示順は日付降順（新しい順）
- [ ] クリックで詳細画面に遷移できる
- [ ] エントリ0件の月は「該当する日記がありません」と表示される
- [ ] オフラインで動作する
- [ ] 外部通信なし、依存追加なし
- [ ] console.logに本文・要約を出さない
