# 実装サイクル計画（Cycle 9 / 0.5日）

## 期間
- （例）2026/02/15 〜 2026/02/15

## 今回のテーマ
PoCを「完成」と言える状態にクローズする。  
機能追加はしない。ドキュメントと判断材料を揃え、次の意思決定を可能にする。

## ゴール（PoCとしての完成）
- READMEを読めば「何ができるPoCか」「何をしないか」が即わかる
- セキュリティ/プライバシー（外部送信なし・暗号化バックアップ）の前提が明文化されている
- 動作確認の手順が揃っている（開発者が再現できる）
- 次フェーズに進む場合の拡張余地が整理されている（境界が明確）

## やらないこと
- 新機能追加（週/月要約、タグ、WebGPU/LLM本格導入、同期など）
- デザイン作り込み
- 依存追加・外部通信追加

## 作成/更新するドキュメント（Must）
1) README.md（更新）
- 1段落の概要（このPoCが何か）
- 主要機能（箇条書き）
- プライバシー方針（外部送信しない、ローカル保存）
- バックアップ（通常/暗号化）について短く
- セットアップ（npm install / dev / build / preview）
- 動作確認チェックリスト（最小）
- スコープ外（やらないこと）を明記

2) docs/DECISIONS.md（追記）
- ローカル保存採用理由
- 外部送信しない設計の理由
- 要約はルールベースをデフォルトにした理由（端末差/安定性）
- 暗号化バックアップ（AES-GCM + PBKDF2）採用理由と注意点

3) playbook/（最終メモ）
- playbook/ARCHITECTURE.md は変更不要（必要なら小さな追記のみ）
- playbook/RELEASE.md（または同等がなければ新規作成してもよい）
  - Pagesへのデプロイ方法（PoC用の最短手順）
  - CSP/ヘッダの最低限（既にあるなら参照だけでもOK）

## 受け入れ条件（DoD）
- [ ] READMEを読めばPoCの価値と境界が理解できる
- [ ] 新規開発者が手順通りに起動できる
- [ ] プライバシー/バックアップ/暗号化の注意点が明文化されている
- [ ] 既存機能が壊れていない（簡易チェックでOK）
